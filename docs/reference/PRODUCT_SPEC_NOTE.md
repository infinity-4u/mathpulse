## How to use it
PRINCIPLES SOURCE ONLY. Do not treat it as the product spec for this repo.

The following UX principles have been extracted and folded into `spec.md` (implemented without runtime AI, per CONTRACT.md §4):
- No-shame repair states — incorrect answers enter a progressive hint/repair flow, never a failure screen
- Mistake taxonomy — deterministic rules classify error type (conceptual, procedural, careless) to select the right repair path
- Parent "tonight's 3 questions" plan — a lightweight parent-visible practice nudge
- Calm mode — reduced visual stimulation option for focus or anxiety
- Accessibility rules — minimum touch targets, contrast, screen reader, dyslexia-friendly font option
